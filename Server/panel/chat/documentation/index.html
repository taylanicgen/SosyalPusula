Requirements

  1. PHP 4

Setup

  1. Open the SquareLC folder and make a copy of the _config.php file called config.php and set your settings in that file
  2. Upload the SquareLC folder on a web hosting of your choice
  3. Give write permissions to the /SquareLC/db/ folder using an FTP client or a file browser your hosting provides (usually chmod 755 or 777 if that doesn’t work)

Configuration

secret
Set this to a long, random string that couldn't be guessed by anyone. This is used for encryption and securing cookies.

tracking
Set this to either "cookie" or "ip". When tracking by cookie, each browser is handled as a separate user, otherwise each IP is. Note that this must be set to "cookie" if you plan on using your own user authentication system to set the nicknames and other data of users in the chat.

log
Set this to either true or false. When set to true, messages will be saved in a text file and will always be shown in the chat window. This can also be numerical to indicate the amount of lines to be shown in the chat window.

static_path
This is the absolute path to the contents of the SquareLC folder where the config.php file is. For example, if the folder is in the root directory of your server, the accepted values would be either "http://yourdomain.com/SquareLC/" or "/SquareLC/". Note that there must be a trailing slash.

profile_path
When not set to false, this is the link that you are redirected to when you click on a username with an id in the chat. You can use {id} and {name} to make profile links for every user. Example: "http://yourdomain.com/user_profile.php?id={id}&name={name}".

load_lines
The time in milliseconds to wait before fetching for new messages. You can make the chat more responsive by lowering this, but it might cause a heavy load on your server.

load_online
The time in milliseconds that a user is reported to have left after being away. Setting this very low might cause two messages (user left, user joined) to appear when a user refreshes the page.

send_timeout
How long in milliseconds a user has to wait before sending another message.

max_users
How many users are allowed in a chat. Set this to false to allow unlimited users.

msg_max_length
The maximum length of a message in chat.

name_min_length
The minimum length of a nickname.

name_max_length
The maximum length of a nickname.

name_case_sensitive
Set this to either true or false. When set to true, the nicknames Test and test (example) will be considered the same.

commands
Here, you can set the level requirement for each command. Set the value to false if you want to disable the command entirely.

users
If you want to make some nicknames password-protected and give them administrative rights, you can set them in this array. The default level is 0.

Including the chat on a page

To use Square Live Chat on a PHP page, you must first include the main SquareLC file (SquareLC.php).

require 'SquareLC/SquareLC.php';

The next step is to select a channel (skipping this step will default to the channel "global"). Channels are essentially chat rooms, users in one channel won't see the messages posted in another one.

SquareLC::init('channel_name');

If you've already selected the channel, but for some reason want to change it, use the channel method:

SquareLC::channel('other_channel');

Now you're ready to include a chat on your page. Here's an example with optional settings:

$settings = array
(
// This is the message on the top of the chat (set this to false or '' to remove the top bar, defaults to 'Live chat')
'title' => 'Chat room one',
// You can set these in either pixels or percents (default to 100%)
'width' => 650,
'height' => 400,
// Set this to true if you want the chat to be full screen, don't output any HTML on the page while using this (defaults to false)
'fullscreen' => false,
// Set this to false if you don't want to show the list of online users (defaults to true)
'view_online' => true,
// Set this to false if you want to remove the outer border of the chat (defaults to true)
'outer_border' => true,
// Set this to true if you want to show a send button next to the message input (defaults to false)
'view_send_btn' => true
);

SquareLC::chat($settings);

Of course, you don't have to set any settings at all. If you're happy with the default values, simply use:

SquareLC::chat();

 

User System Integration

If you have a user system, you can make the nickname in chat equal to the name of the logged in user and disable the option to change its nickname. Here's an example on how to do this when the user data in saved in a session:

// First, select the channel
SquareLC::init('channel_name');

// Now, set the user data
SquareLC::user('session', array
(
// You MUST set an id for the user if you are using a user system
'id' => $_SESSION['user_id'],
// Setting lock to true prevents the user from changing their nickname
'@lock' => true
'@name' => $_SESSION['user_name'],
));

// You can also increase the user's level and enable more commands if it's an administrator or moderator
if($_SESSION['user_group'] == 'administrator')
{
SquareLC::user('Session', array
(
'@level' => 3
));
}

The @ in front of the key only sets the value in the selected channel, never use it for the id.

Language

To translate the site, open the SquareLC/lang/ folder and make a copy of the en.php file. Call it the ISO code of your language (de for German, fr for France, ...) and make translations in that file. It's not recommended to directly modify the en.php file because it may be overwritten by an update.

Once you've finished making the translations, open config.php in the SquareLC folder and set lang equal to your language code.

Style

You can change the appearance of the chat using CSS. Open the SquareLC/theme/ folder and make a copy of the default.css file, call it the name of your new theme. You should also look at the SquareLC/css/layout.css file if you want to change the layout. The CSS rules in your theme file will overwrite the ones in SquareLC/css/layout.css.

To activate the new theme, open config.php in the SquareLC folder and set style equal to the name of your theme.
